## Usage
<hr />

```javascript
import ToastButton from "@/components/uploadthingui/toast-button";
```

```javascript
<ToastButton
  props={{ endpoint: "imageUploader" }}
  instanceId="toast-button-general"
  showDetails
/>
```

Make sure to add the `sonner` inside your `layout.tsx` as well

````javascript
import { Toaster } from 'sonner';
```

```javascript
<Toaster />
````

<br />
## Props
<hr />

| Name                  | Type         | Required | Default | Description                                                                                                                                                 |
| --------------------- | ------------ | -------- | ------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| props                 | uploadthing  | yes      | -       | Accepts all the props that are available with [uploadthing](https://docs.uploadthing.com/api-reference/react#props) <br/><br/> Only the `endpoint` required |
| showDetails           | boolean      | no       | false   | Show the details/metadata for the upload route endpoint                                                                                                     |
| instanceId            | string       | yes      | -       | This will be used as an identifier if multiple file upload buttons are used at the same time <br/><br/> Better to add to avoid duplication of file uploads  |
| allowInBetweenUploads | boolean      | no       | true    | Whether to allow in between uploads <br/><br/> If `false` then when files are being uploaded, no new files can be uploaded.                                 |
| routeDetails          | routeDetails | no       | -       | Additional details for the upload; Example: maxFileCount, minFileCount, etc. <br/><br/>Recommended to add these details here instead of your route.         |
| children              | ReactNode    | no       | -       | Any node written will take the place of the action button                                                                                                   |

### Route Details
<hr />

<Note title="Note" type="danger">
Add route details (`minFileCount` & `maxFileCount`) here and not inside your
`core.ts`. Reason is because as each file upload runs its own `startUpload`.

If a route has a `minFileCount` set to `2` then that condition will never be
hit on route level. Thus you shouldn't add both of these properties inside
your `core.ts`, instead add them at the component level.

</Note>

<hr />

<Note title="Note" type="note">
  Set to `true` by default. When set to `false` you won't be able to upload any
  files when a file is being uploaded.
